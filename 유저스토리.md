# GemmeNews 유저스토리

## 프로젝트 목표
여러 뉴스 사이트의 기사를 한곳에 모아서 todo 앱처럼 관리할 수 있는 뉴스 애그리게이터 서비스를 제공한다.

## 사용자 역할
- **사용자**: 뉴스를 읽고 관리하는 사람

---

## Epic 1: 뉴스 수집 시스템

### US-1.1: 기본 뉴스 소스 수집
**As a** 시스템
**I want to** 하루에 한 번 정해진 뉴스 소스들을 자동으로 수집한다
**So that** 사용자들이 최신 뉴스를 볼 수 있다

**수집 대상:**
- Hacker News (https://news.ycombinator.com/)
- Geek News (https://news.hada.io/)
- 네이버 뉴스 경제면 (https://news.naver.com/section/101)

**인수 조건:**
- [ ] 각 뉴스 소스에서 기사 제목과 링크를 추출한다
- [ ] 각 기사의 원문 내용을 수집한다
- [ ] 하루에 한 번 실행되는 엔드포인트를 통해 수집이 진행된다
- [ ] 이미 수집된 기사는 중복으로 수집하지 않는다
- [ ] 수집된 날짜/시간이 기록된다

**개발자 노트:**
- 뉴스 소스는 코드/설정 파일로 관리된다
- 각 소스별로 다음 정보를 정의해야 한다:
  - 소스 이름 및 기본 URL
  - 목록 페이지의 CSS 선택자 (기사 제목, 링크)
  - 상세 페이지의 CSS 선택자 (본문 내용)
- 새로운 뉴스 소스 추가는 개발자가 코드를 수정해서 배포한다
- 향후 Epic 5에서 일반 사용자도 UI로 추가할 수 있도록 확장 예정

## Epic 2: 사용자 뉴스 읽기

### US-2.1: 뉴스 피드 보기
**As a** 사용자
**I want to** 오늘 수집된 뉴스 목록을 한눈에 볼 수 있다
**So that** 어떤 뉴스들이 있는지 파악할 수 있다

**인수 조건:**
- [ ] 읽지 않은 뉴스 목록이 기본으로 표시된다
- [ ] 각 뉴스 항목에 제목과 출처가 표시된다
- [ ] 뉴스를 클릭하면 상세 내용을 볼 수 있다
- [ ] 뉴스 소스별로 필터링할 수 있다
- [ ] 날짜별로 뉴스를 볼 수 있다

### US-2.2: 뉴스 상세 내용 읽기
**As a** 사용자
**I want to** 이메일 클라이언트처럼 뉴스를 선택하면 옆에서 내용을 볼 수 있다
**So that** 편하게 뉴스를 읽을 수 있다

**인수 조건:**
- [ ] 목록에서 뉴스를 클릭하면 같은 화면에서 내용이 표시된다
- [ ] 뉴스 제목, 출처, 수집 날짜가 표시된다
- [ ] 본문 내용이 읽기 좋게 표시된다
- [ ] 원문 링크로 이동할 수 있다
- [ ] 키보드로 이전/다음 뉴스로 이동할 수 있다

## Epic 3: 뉴스 관리

### US-3.1: 뉴스 완료 처리
**As a** 사용자
**I want to** 읽은 뉴스를 완료(아카이브) 처리할 수 있다
**So that** todo 앱처럼 읽은 뉴스와 안 읽은 뉴스를 구분할 수 있다

**인수 조건:**
- [ ] 뉴스 항목마다 완료 버튼이 있다
- [ ] 완료 처리하면 기본 목록에서 사라진다
- [ ] 완료된 뉴스는 별도 아카이브 섹션에서 볼 수 있다
- [ ] 완료를 취소할 수 있다
- [ ] 완료된 뉴스 개수가 표시된다

### US-3.2: 뉴스 관심 없음 처리
**As a** 사용자
**I want to** 관심 없는 뉴스를 빠르게 제거할 수 있다
**So that** 읽고 싶은 뉴스에만 집중할 수 있다

**인수 조건:**
- [ ] 뉴스 항목마다 관심 없음 버튼이 있다
- [ ] 관심 없음 처리하면 목록에서 즉시 사라진다
- [ ] 관심 없음 처리된 뉴스는 다시 표시하지 않는다
- [ ] 실수로 누른 경우 되돌릴 수 있는 Undo 기능이 있다 (짧은 시간 동안)

### US-3.3: 뉴스 리마인드 설정
**As a** 사용자
**I want to** 나중에 읽고 싶은 뉴스의 리마인드를 설정할 수 있다
**So that** 지금은 시간이 없지만 나중에 꼭 읽을 수 있다

**인수 조건:**
- [ ] 뉴스 항목마다 리마인드 버튼이 있다
- [ ] 리마인드 옵션을 선택할 수 있다:
  - 1시간 후
  - 3시간 후
  - 내일
  - 모레
- [ ] 리마인드 설정하면 목록 상단에 고정되거나 별도 섹션에 표시된다
- [ ] 리마인드 시간이 지나면 다시 읽지 않음 목록 최상단에 표시된다
- [ ] 리마인드를 취소할 수 있다

## Epic 4: 뉴스 공유

### US-4.1: 뉴스 링크 공유
**As a** 사용자
**I want to** 읽은 뉴스를 다른 사람과 공유할 수 있다
**So that** 흥미로운 뉴스를 친구나 동료에게 쉽게 전달할 수 있다

**인수 조건:**
- [ ] 뉴스 상세 화면에 공유 버튼이 있다
- [ ] 공유 버튼을 누르면 공유 링크가 클립보드에 복사된다
- [ ] 복사 완료 메시지가 표시된다
- [ ] 공유 링크는 짧고 깔끔한 형식이다 (예: /share/[news-id])

### US-4.2: 공유 링크 프리뷰
**As a** 뉴스를 공유받은 사람
**I want to** SNS나 메신저에서 링크를 붙여넣었을 때 제목과 내용 미리보기가 표시된다
**So that** 링크를 클릭하기 전에 어떤 내용인지 알 수 있다

**인수 조건:**
- [ ] Open Graph 메타 태그가 포함된다:
  - og:title (뉴스 제목)
  - og:description (뉴스 요약 또는 첫 부분)
  - og:url (공유 링크 URL)
  - og:type (article)
  - og:site_name (GemmeNews)
- [ ] Twitter Card 메타 태그도 포함된다
- [ ] 공유 페이지는 서버 사이드 렌더링(SSR)으로 처리된다
- [ ] 공유 링크로 접속하면 해당 뉴스 상세 내용을 볼 수 있다
- [ ] 비로그인 사용자도 공유된 뉴스를 읽을 수 있다

**개발자 노트:**
- SolidStart의 SSR 기능을 활용해 공유 페이지를 서버에서 렌더링
- 각 공유 페이지마다 동적으로 메타 태그를 생성
- SEO와 소셜 미디어 크롤러를 고려한 구현

## Epic 5: 사용자 계정

### US-5.1: 회원가입
**As a** 신규 사용자
**I want to** 계정을 생성할 수 있다
**So that** 내 뉴스 읽기 진행상황을 저장하고 관리할 수 있다

**인수 조건:**
- [ ] 이메일과 비밀번호로 가입할 수 있다
- [ ] 이메일 중복을 확인한다
- [ ] 비밀번호는 안전하게 저장된다

### US-5.2: 로그인/로그아웃
**As a** 사용자
**I want to** 로그인하고 로그아웃할 수 있다
**So that** 내 계정에 접근하고 보안을 유지할 수 있다

**인수 조건:**
- [ ] 이메일과 비밀번호로 로그인할 수 있다
- [ ] 로그인 상태가 유지된다 (세션/토큰)
- [ ] 로그아웃하면 모든 인증 정보가 제거된다
- [ ] 로그인하지 않으면 뉴스 목록에 접근할 수 없다

## 기술 스택 참고

- **프론트엔드/백엔드**: Solid Start
- **데이터베이스**: Drizzle + PostgreSQL
- **API**: Pothos (GraphQL) + Drizzle plugin
- **수집 실행**: 백엔드 엔드포인트 (별도 인프라 없음)

## 화면 구성 참고

메인 화면은 이메일 클라이언트 스타일:
- 왼쪽: 뉴스 목록 (제목, 출처, 액션 버튼)
- 오른쪽: 선택한 뉴스의 상세 내용
- 상단: 필터 및 정렬 옵션
